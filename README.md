# 📈 Stock Radar — A股实时推送雷达

每个 A 股交易日（自动过滤节假日），按固定时间窗口抓取个股行情、资金流向、K 线技术面，结合大盘情绪与行业/概念板块资金动向，以**飞书消息卡片**形式推送到飞书群机器人。

**新增**：每日晨间要闻推送（08:00，含非交易日）+ 全球市场收盘行情推送 + 飞书消息卡片 + 30分钟MACD分析。

---

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 📅 节假日过滤 | 接入新浪 A 股交易日历，自动跳过节假日（仅A股分析） |
| 📊 个股分析 | 实时行情 + K 线均线/RSI/量比 + 主力资金流向 |
| 📉 30分钟MACD | 当日30分钟K线 MACD 金叉/死叉判断，T+0 操作方向参考 |
| 🏛️ 大盘概况 | 上证/深证/创业板/科创50/沪深300/北证50 |
| 💰 资金流向板块 | 主力净流入最多的行业/概念 Top3 |
| 🔥 成交活跃板块 | 成交金额最高的行业/概念 Top3 |
| 🌅 晨间要闻 | 每日08:00推送影响A股的财联社实时快讯（含非交易日） |
| 🌐 全球市场 | 美股/日本/港股/台湾/越南/印度收盘后30分钟推送 |
| 🃏 飞书卡片 | Interactive Card 格式：涨红跌绿橙色强势青色晨报，视觉清晰 |
| 🛡️ 多重兜底 | 东财接口 → 新浪备用，卡片推送失败自动降级纯文本 |
| 🔧 灵活配置 | 增减股票只需编辑 `config.py` |

## 📁 项目结构

```
stock_radar/
├── config.py        # 配置：股票列表、推送时间、Webhook、签名密钥

├── requirements.txt
└── README.md
```

## 🚀 快速开始

### 1. 环境要求

- Python 3.12+
- macOS / Linux

### 2. 安装依赖

```bash
cd stock_radar
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. 配置

编辑 `config.py`：

```python
# 目标股票（随时增删）
TARGET_STOCKS = [
    {"code": "600389", "name": "江山股份",   "market": "SH"},
    {"code": "588190", "name": "科创100ETF", "market": "SH"},
    {"code": "588080", "name": "科创50ETF",  "market": "SH"},
]

# 飞书群机器人 Webhook
FEISHU_WEBHOOK = "https://open.feishu.cn/open-apis/bot/v2/hook/YOUR_HOOK_ID"

# 飞书签名密钥（不需要则置空字符串）
FEISHU_SECRET = "your_secret"
```

### 4. 启动

```bash
bash run.sh
# 或
source .venv/bin/activate && python main.py
```

启动后会立即执行一次分析推送，然后按 `PUSH_TIMES` 定时运行。

### 5. macOS 开机自启（LaunchAgent）

```bash
# 安装 plist（已内置，改好路径后执行）
launchctl load ~/Library/LaunchAgents/com.stockradar.plist

# 停止
launchctl unload ~/Library/LaunchAgents/com.stockradar.plist

# 查日志
tail -f ~/openclawdir/stock_radar/stock_radar.log
```

## 🃏 飞书消息卡片说明

推送升级为飞书 **Interactive Card**，视觉效果大幅提升：

### 颜色规则

| Header 颜色 | 触发条件 |
|------------|---------|
| 🔴 红色（red） | 整体偏涨 |
| 🟢 绿色（green） | 整体偏跌 |
| 🟠 橙色（orange） | 强势上涨（涨幅 >3%） |
| 🩵 青色（turquoise） | 晨间要闻 / 全球市场推送 |
| 🔵 蓝色（blue） | 默认 / 无法判断方向 |

### 卡片结构

- 每个 `━━━ xxx ━━━` 分隔块 → 一个独立 `div` 元素
- 块与块之间自动插入 `hr` 分隔线
- 支持 **粗体**、换行等 lark_md 格式

### 降级策略

卡片推送失败（网络错误、格式问题）时，自动降级为纯文本推送，不影响信息送达。

## 📉 30分钟MACD说明

每次个股分析报告末尾新增一行 `📉 30分钟MACD：xxx`，辅助盘中 T+0 操作判断。

### 数据来源

- 主渠道：`stock_zh_a_hist_min_em(period="30")`（东财）
- ETF 备用：`fund_etf_hist_min_em(period="30")`
- 仅获取**当日**数据

### MACD 参数

- 快线 EMA：12
- 慢线 EMA：26
- 信号线 DEA：9

### 输出结论示例

| 场景 | 输出 |
|------|------|
| DIF 从下方穿越 DEA（零轴上方） | `30分钟MACD金叉（多头区），短线多头信号` |
| DIF 从下方穿越 DEA（零轴下方） | `30分钟MACD金叉（空头区），短线多头信号` |
| DIF 从上方穿越 DEA（零轴上方） | `30分钟MACD死叉（多头区），注意回调` |
| DIF 从上方穿越 DEA（零轴下方） | `30分钟MACD死叉（空头区），注意回调` |
| DIF > DEA，DIF > 0 | `30分钟MACD多头运行，趋势偏强` |
| DIF < DEA，DIF < 0 | `30分钟MACD空头运行，谨慎偏空` |
| 数据不足（< 5 根） | `30分钟数据不足` |

> ⚠️ 盘前（09:15 首次推送）30分钟数据不足属正常，随着日内K线积累会越来越准确。

## 🛡️ 兜底策略

| 数据类型 | 主渠道 | 备用渠道 |
|---------|--------|---------|
| 个股实时行情 | 新浪 `hq.sinajs.cn`（今日实时） | 东财 `stock_zh_a_spot_em` |
| ETF 实时行情 | 新浪 `hq.sinajs.cn` | 东财 `fund_etf_spot_em` |
| 日线K线 | `stock_zh_a_hist`（东财） | `stock_zh_a_daily`（新浪） |
| ETF日线K线 | `stock_zh_a_hist`（东财） | `fund_etf_hist_sina`（新浪） |
| 30分钟K线 | `stock_zh_a_hist_min_em`（东财） | `fund_etf_hist_min_em`（ETF专用） |
| 大盘指数 | `stock_zh_index_spot_em`（东财） | `stock_zh_index_spot_sina`（新浪） |
| 晨间要闻 | `stock_info_global_cls`（财联社） | `stock_news_main_cx`（财新） |
| 全球股指 | `index_global_spot_em`（东财） | — |
| 资金流向 | `stock_individual_fund_flow`（东财） | ⚠️ 无备用，失败返回空 |
| 飞书推送 | Interactive Card | 自动降级纯文本 |

所有接口均有 `try/except` 兜底，单点失败不影响整体推送。

## ➕ 增减股票

只需修改 `config.py` 中的 `TARGET_STOCKS`：

```python
TARGET_STOCKS = [
    {"code": "600389", "name": "江山股份",   "market": "SH"},
    {"code": "588190", "name": "科创100ETF", "market": "SH"},
    {"code": "588080", "name": "科创50ETF",  "market": "SH"},
    # 添加新股票 ↓
    {"code": "000001", "name": "平安银行",   "market": "SZ"},
]
```

修改后重启程序生效。
